This is a security feature that prevents their site from being displayed in frames.

‚úÖ Solution 1: Extract Direct Stream URL RECOMMENDED

Get the direct HLS/m3u8 stream URL and play it without the website wrapper.

‚úÖ Pros:

  • No ads
  • No X-Frame-Options issues
  • Lightweight playback
  • Full control

‚ùå Cons:

  • Requires yt-dlp
  • URL may change

For Raspberry Pi:

# Install yt-dlp
pip3 install yt-dlp

# Extract stream URL
STREAM_URL=$(yt-dlp -g "https://rtmklik.rtm.gov.my/live/tv2")

# Play with omxplayer (recommended)
omxplayer --live "$STREAM_URL"

# Or with VLC
cvlc "$STREAM_URL" --fullscreen

For Windows (Testing):

# Install yt-dlp
pip install yt-dlp

# Get URL
yt-dlp -g "https://rtmklik.rtm.gov.my/live/tv2"

# Play with VLC
vlc "PASTE_URL_HERE"

üåê Solution 2: Open in Direct Window

Open RTM site directly in Chromium (not in iframe).

‚úÖ Pros:

  • Simple to implement
  • No URL extraction needed
  • Official player

‚ùå Cons:

  • May show ads
  • Full webpage overhead
  • Harder to control

For Raspberry Pi:

chromium-browser --kiosk \
  --autoplay-policy=no-user-gesture-required \
  --disable-infobars \
  "https://rtmklik.rtm.gov.my/live/tv2"

üîÑ Solution 3: Use CORS Proxy ADVANCED

Run a local proxy that strips X-Frame-Options header.

‚úÖ Pros:

  • Can embed in iframe
  • Works with any site

‚ùå Cons:

  • Complex setup
  • Requires proxy server
  • May violate ToS
  • Legal concerns

Example (for development only):

# Using CORS Anywhere
git clone https://github.com/Rob--W/cors-anywhere.git
cd cors-anywhere
npm install
node server.js

# Then access via: http://localhost:8080/https://rtmklik.rtm.gov.my/live/tv2

üé¨ Solution 4: HTML5 Video Element

If you have the direct stream URL, use HTML5 video player.

<video id="stream" controls autoplay>
  <source src="STREAM_URL.m3u8" type="application/x-mpegURL">
</video>

<script src="https://cdn.jsdelivr.net/npm/hls.js@latest"></script>
<script>
  if (Hls.isSupported()) {
    var video = document.getElementById('stream');
    var hls = new Hls();
    hls.loadSource('STREAM_URL.m3u8');
    hls.attachMedia(video);
  }
</script>
